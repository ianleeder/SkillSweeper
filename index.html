Skillsweeper

Minesweeper, without the luck.

View on GitHub Download .zip Download .tar.gz

Purpose

I love playing MineSweeper, but I hate the random chance that detracts from the skill of playing the game. So I set out to make a new version of MineSweeper that would remove the element of chance, leaving just a game of skill: SkillSweeper.

I also wanted to challenge myself, so I wrote it exclusively in Javascript, with no external libraries.

How it works

SkillSweeper runs a detection algorithm to see if there are any moves which can be logically deduced to be safe or mines. If it detects that none of these are available, and you make a move which results in death, the game does not count against your record: you lost due to bad luck. Win/loss percentage is an indicator of skill.

There is also a HINT button. If you click the hint button, one of two things will happen.

  1. If no available moves can be detected, it will tell you so. Or,
  2. It will show you a move at the cost of 10 seconds being added to the clock. A cell will be highlighted green (safe to reveal), or red (guaranteed mine and should be flagged).
Please note this algorithm is not intended to be smarter than an expert player. The object was not to create Deep Blue for MineSweeper. The intent is merely to remove the element of luck, by detecting a reasonable amount of safe moves. It does not take probability into account, or the remaining amount of mines.

Status

This is a pretty raw alpha version of the application.

Implemented

  • Basic detection of safe cells and certain mines
  • Basic game state
  • Autoplay mode
  • Game win detections
  • More advanced detection algorithm
  • Timer
  • Remaining mines counter
